## Benchmark Plan — 2026-04-15

### Selected Experiments (rank by expected ROI)
For each experiment (2-5 total):

#### 1. vLLM Gemma 4 + TurboQuant — vLLM
**Hypothesis**: The combination of Gemma 4 architecture support and TurboQuant should significantly improve inference throughput and reduce memory usage.
**Setup**:
  - Model: Llama-3-70B (fits in 128GB/node)
  - Framework version: `vLLM` (latest tag that includes the merged PRs for Gemma 4 and TurboQuant support)
  - Node config: Both nodes (TP=2 across nodes)
  - Parallelism: Tensor Parallelism (TP=2) and Pipeline Parallelism (PP=1)
**Benchmark script**:
```bash
# Update repository and setup environment
cd ~/inference-bench-env
git clone https://github.com/vllm-project/vllm.git
cd vllm
git pull
git checkout develop

# Install dependencies
python -m pip install -r requirements.txt
cd ../

# Create virtual environment
python -m venv infer-bench-venv
source infer-bench-venv/bin/activate

# Install specific version of transformers
pip install transformers>=5.5.0

# Prepare benchmarking scripts and data
mkdir -p benchmarks
cd benchmarks
git clone https://github.com/qwen-inference/benchmark-data.git

# Run benchmark
export CUDA_VISIBLE_DEVICES=0,1
cd ../../vllm
python scripts/benchmarks/llamav3.py --model-path /path/to/llama-3-70b --tokenizer-path /path/to/llama-3-70b --max-new-tokens 128 --batch-size 8 --num-warmup-iter 10 --num-iter 100

# Deactivate and exit virtual environment
deactivate
```
**Metrics to capture**: TTFT (ms), TPOT (ms/tok), throughput (tok/s), GPU memory (GB), MFU (%)
**Baseline**: vLLM without Gemma 4 and TurboQuant support
**Expected delta**: 20-30% improvement in throughput, 20-30% reduction in GPU memory usage, and 10-15% improvement in MFU.
